# 🌴 نظام إدارة الاستراحة

نظام ويب متكامل لإدارة استراحة مملوكة لثلاثة شركاء — **جلند، سعيد، سالم** — يتولّى توزيع الأيام، جدولة الصيانة الأسبوعية، تبادل الأيام بين الشركاء، والإحصائيات. مبني بـ HTML5 و CSS3 و JavaScript (ES6+) **بدون أي مكتبات خارجية**، ويحفظ بياناته محلياً مع بنية جاهزة للربط بقاعدة بيانات سحابية لاحقاً.

---

## ✨ المزايا

- **تسجيل دخول لكل شريك** بكلمة مرور خاصة، مع جلسة محفوظة.
- **تقويم شهري** يوضّح مالك كل يوم بلون مميّز.
- **صيانة دورية كل أربعاء**، ومسؤول الصيانة يتناوب أسبوعياً بين الشركاء، بتمييز بصري واضح (بلاطة داكنة بإطار ذهبي).
- **بانر "اليوم"** يعرض فوراً صاحب الاستراحة (أو مسؤول الصيانة) لليوم الحالي.
- **لوحة تحكم (Dashboard)**: عدد أيام كل شريك في الشهر، عدد مرات الصيانة، إحصاء سنوي، ورسوم بيانية (أعمدة + دائرة) مرسومة يدوياً بلا مكتبات.
- **تبادل الأيام**: يطلب الشريك تبادل يوم، يصل الطرف الآخر إشعار، ويقبل أو يرفض — ويُحدَّث الجدول تلقائياً عند القبول.
- **سجل تاريخي** لكل العمليات (طلبات، قبول، رفض، تعديلات النظام).
- **إشعارات**: طلب تبادل وارد، تنبيه قبل يوم استراحتك بيوم، تنبيه مسؤولية الصيانة الأسبوعية. مع دعم اختياري لإشعارات المتصفح.
- **إعدادات**: تغيير كلمة المرور، اختيار اللون المميّز، تفعيل/تعطيل الإشعارات.
- **تصدير/استيراد JSON** للنسخ الاحتياطي ونقل البيانات بين الأجهزة.
- **تصميم متجاوب** للهاتف واللوحي والكمبيوتر، يدعم RTL والوضع المخفّف للحركة.

## 🚀 طريقة التشغيل

### محلياً
بما أن المشروع يستخدم عدة ملفات JS، يُفضّل تشغيله عبر خادم محلي (لتجنّب قيود `file://`):

```bash
# داخل مجلد المشروع
python3 -m http.server 8080
# ثم افتح: http://localhost:8080
```

أو باستخدام إضافة **Live Server** في VS Code.

> يعمل أيضاً بفتح `index.html` مباشرة في أغلب المتصفحات، لكن بعض الميزات (مثل manifest الـ PWA) تتطلب خادماً.

### كلمات المرور الافتراضية
| الشريك | كلمة المرور |
|--------|-------------|
| جلند   | `Juland2026` |
| سعيد   | `Said2026`   |
| سالم   | `Salim2026`  |

يمكن تغييرها من صفحة الإعدادات بعد الدخول.

## 🌐 الرفع على GitHub Pages

1. أنشئ مستودعاً جديداً على GitHub وارفع محتويات المجلد بالكامل:
   ```bash
   git init
   git add .
   git commit -m "نظام إدارة الاستراحة"
   git branch -M main
   git remote add origin https://github.com/<اسم-المستخدم>/<اسم-المستودع>.git
   git push -u origin main
   ```
2. من المستودع: **Settings → Pages**.
3. تحت *Build and deployment* اختر **Deploy from a branch**، ثم الفرع `main` والمجلد `/ (root)`، واحفظ.
4. بعد دقائق سيكون الموقع متاحاً على:
   `https://<اسم-المستخدم>.github.io/<اسم-المستودع>/`

> كل المسارات في المشروع **نسبية** (`css/…`, `js/…`, `assets/…`)، لذا يعمل مباشرة على GitHub Pages دون تعديل.

## 🗂 الهيكل البرمجي

```
/
├── index.html              # الهيكل وكل العروض (تقويم/لوحة/سجل/إعدادات)
├── manifest.json           # بيانات PWA (للتثبيت المستقبلي)
├── README.md
├── css/
│   └── style.css           # كل التنسيقات (هوية بصرية + تجاوب RTL)
├── js/
│   ├── storage.js          # طبقة تخزين موحّدة (localStorage الآن، Backend لاحقاً)
│   ├── auth.js             # تسجيل الدخول والجلسة وكلمات المرور
│   ├── calendar.js         # منطق الجدولة والصيانة وعرض التقويم
│   ├── notifications.js    # توليد الإشعارات
│   └── app.js              # المتحكّم الرئيسي وربط كل الوحدات
└── assets/
    └── icon.svg            # أيقونة التطبيق
```

### مبدأ الجدولة (قابل للتدقيق)
- يوم الراحة: `partners[totalDays % 3]` حيث `totalDays` عدد الأيام منذ مبدأ التوقيت.
- الصيانة: كل **أربعاء**، ومسؤولها `partners[floor(totalDays / 7) % 3]` (تناوب أسبوعي).
- أي تبادل مقبول يُحفظ كـ `override` يَحُل محل المالك المحسوب لذلك اليوم فقط.

## 🔮 التوسّع المستقبلي

صُمّم الكود ليكون قابلاً للترقية دون إعادة كتابة:

- **قاعدة بيانات سحابية / مزامنة بين الأجهزة**: كل عمليات البيانات تمرّ عبر `js/storage.js` ودوالّه `async`. للربط بـ **Firebase** أو **Supabase** أو **REST API**، عدّل فقط أجسام الدوال `read / write / removeKey` داخل هذا الملف، وابقِ التواقيع كما هي — لن تحتاج لتغيير بقية الملفات.
- **PWA**: ملف `manifest.json` جاهز. لتفعيل العمل دون اتصال، أضِف `service-worker.js` وسجّله في `app.js`.
- **شركاء إضافيون أو قواعد مختلفة**: عدّل قائمة `DEFAULT_PARTNERS` في `storage.js` ومنطق الدورة في `calendar.js`.
